# Website Management Flow

```mermaid
graph TD
    A["Start: Need to update website data docs"] --> B{"What is the goal?"}
    B -->|Add new dataset| C1["Run manage_dataset_docs add"]
    C1 --> C2["Fill prompts (metadata and paths)"]
    C2 --> C3["Review scaffolded overview and validation report"]
    C3 --> C4{"Validation ready?"}
    C4 -->|Yes| C5["Commit docs generated by CLI"]
    C4 -->|No| C6["Adjust conversion or ranges"]
    C6 --> C1

    B -->|Refresh validation| E1["Run manage_dataset_docs refresh-validation"]
    E1 --> E2["create_dataset_validation_report (no plots)"]
    E2 --> E3["Inspect text report and links"]
    E3 --> E4{"Ranges require tuning?"}
    E4 -->|Yes| E5["Use interactive_validation_tuner"]
    E5 --> E6["Update custom ranges YAML"]
    E6 --> E1
    E4 -->|No| E7["Commit validation report updates"]

    B -->|Edit metadata or download links| D1["Run manage_dataset_docs edit-metadata"]
    D1 --> D2["Review CLI-updated registry and docs"]
    D2 --> D3{"Need validation refresh?"}
    D3 -->|Yes| E1
    D3 -->|No| D5["Commit CLI changes"]

    B -->|Retire dataset| F1["Run manage_dataset_docs retire"]
    F1 --> F2["Review archived/flagged docs"]
    F2 --> F3["Commit changes"]

    D5 --> G["End"]
    C5 --> G
    E7 --> G
    F3 --> G
```

Use this flow as a quick reference before editing dataset documentation. The CLI prompts for metadata and updates the registry automaticallyâ€”no manual YAML edits required.
